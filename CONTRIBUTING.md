# Contributing

Thanks for helping improve ProDispatch! Please follow these guidelines:

1. Fork the repo and create a feature branch.
2. Keep changes small and focused; include tests.
3. Run dotnet format if you add public APIs.
4. Ensure dotnet test passes for the solution.
5. Open a PR using the template; describe behavior changes and risks.

## Issue Reports
- Include reproduction steps and expected vs actual behavior.
- Share environment details (OS, .NET SDK version).

## Pull Requests
- Update docs when adding features.
- Add benchmarks when performance is relevant.
- Keep console output concise and structured.
